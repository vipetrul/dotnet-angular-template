Philosophy behind template

Target use

  • Intranet applications / not for general public use
  • Desktop web apps
  • Mobile apps (via Cordova or similar)

Goals

  • Maximize real estate, especially on smaller screens (reactive layout)
  • Use material design, because it is not space greedy and looks consistently good on desktop and mobile devices.
  • Reflect impersonated login (login with service account / account must contain "_")

Things to consider

  • Is it OK to use default material design theme? Or should it be customized for UIowa?
  • Does each application should have a logo (ex. Universal Workflow https://workflow.uiowa.edu/inbox )
  • Should we include footer? If so, what should be included there?
